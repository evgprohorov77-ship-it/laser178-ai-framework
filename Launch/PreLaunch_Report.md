## Критические проблемы (P0 — блокеры запуска)

| ID | Проблема | Почему блокер | Что нужно | Владелец |
|----|----------|---------------|-----------|----------|
| P0-1 | **Curl не получает HTML-контент** (нет title, meta, h1, h2, img в исходном HTML) | Поисковые боты не могут проиндексировать сайт. | Проверить кэш/плагины (WP Rocket, LiteSpeed, Elementor lazy render). | Operator |
| P0-2 | **XML sitemap пустые** | Боты не узнают URL сайта. | Перегенерировать sitemap в Yoast SEO. | Operator |
| P0-3 | **Битая ссылка `/bortovoy-zhurnal/` в главном меню** | 404 для пользователей и ботов. | Заменить на `/blog/`. | Operator |
| P0-4 | **Дубли страниц** | Размывание веса, каннибализация. | 301 редирект или удаление дублей. | Operator |
| P0-5 | **Не проверены формы и заявки** | Сайт не может принимать клиентов. | Протестировать форму, чат, телефон. | Owner / Operator |
| P0-6 | **Страница Контакты без адреса и карты** | Нет локального SEO, сложно добраться. | Добавить адрес, проезд, карту. | Owner / Hermes |
| P0-7 | **Страница О нас пустая** | Нет доверия, нет УТП. | Наполнить контентом. | Owner / Hermes |

## Высокие приоритеты (P1)

- Наполнить страницу **Услуги**.
- Добавить фото и описания в **Наши работы**.
- Уточнить **Прайс** реальными ценами.
- Проверить **мобильную версию**.
- Проверить **Core Web Vitals**.
- Добавить **Schema.org LocalBusiness**.

## Средние приоритеты (P2)

- Meta title/description для всех страниц.
- Open Graph теги.
- Alt-теги изображений.
- Canonical.
- Редиректы HTTP → HTTPS (проверить массово).
- Кастомная 404 страница.

## Низкие приоритеты (P3)

- Оптимизация изображений (WebP).
- Search Console / Яндекс.Вебмастер.
- Аналитика.
- Favicon.
- Кроссбраузерность.

## Post-launch (P4)

- Регулярные статьи в блог.
- Аудит битых ссылок.
- Мониторинг Core Web Vitals.
- Мониторинг позиций.
- Добавление отзывов и кейсов.
- **WordPress Adapter** — запрещено до запуска.
- **Новые LAOS Engine** — запрещено до запуска.

## Рекомендуемая дата запуска

**Не ранее устранения всех P0-блокеров.**

- При оперативном доступе к админке/Operator: **3–5 рабочих дней**.
- Без доступа: **блокируется до получения доступа или данных**.

## Рекомендации по следующему шагу

1. **Срочно** получить доступ к админке WordPress или список активных плагинов для диагностики P0-1 и P0-2.
2. Параллельно заполнить `Owner/MASTER_PROFILE.yaml` (адрес, точные цены, гарантия, условия).
3. Подготовить контент для страниц Услуги, О нас, Контакты, Гарантия, Наши работы.
4. После P0-1/P0-2 — запустить SEO-проверку и добавить сайт в Search Console / Вебмастер.

## Резюме для владельца

Сайт визуально работает, контент частично есть, но **не готов к запуску** из-за критических технических проблем с индексацией. Без исправления P0-1 и P0-2 поисковые системы не увидят сайт, и клиенты не будут приходить из поиска. Остальные задачи — контент и UX — решаются быстрее, после получения данных.
