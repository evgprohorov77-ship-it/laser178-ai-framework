# Analytics Checklist

## Версия

1.0.0

## Цель

Можно отслеживать трафик, поведение пользователей и заявки после запуска.

## Легенда

- `[ ]` TODO
- `[>]` IN PROGRESS
- `[!]` BLOCKED
- `[x]` DONE

## Проверки

| ID | Задача | Статус | Владелец | Примечание |
|----|--------|--------|----------|------------|
| A-1 | Подключить Яндекс.Метрику | `[ ]` | Operator | После запуска или до, по согласованию. |
| A-2 | Подключить Google Analytics 4 | `[ ]` | Operator | Опционально. |
| A-3 | Настроить цели на заявки / звонки | `[ ]` | Operator | Кнопки tel:, форма, чат. |
| A-4 | Настроить Google Search Console | `[ ]` | Operator | После фикса sitemap. |
| A-5 | Настроить Яндекс.Вебмастер | `[ ]` | Operator | После фикса sitemap. |
| A-6 | Регулярные отчёты по KPI | `[ ]` | Hermes | Еженедельно после запуска. |
| A-7 | Отслеживание позиций по ключевым запросам | `[ ]` | Hermes | Опционально. |

## Критерии готовности

- [ ] Метрика/аналитика подключена.
- [ ] Цели на конверсию настроены.
- [ ] Search Console и Вебмастер подключены.
- [ ] Есть план регулярных отчётов.

## Риски

- Без аналитики невозможно оценить эффективность запуска.
- Privacy-conscious подход: не передавать персональные данные в аналитику без согласия.
