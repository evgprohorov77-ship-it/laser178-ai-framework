# Чек-лист качества

Перед публикацией любой правки или материала проверяем по этому списку.

## Страница / Пост

- [ ] Один H1 на странице.
- [ ] H2/H3 логически следуют за H1.
- [ ] Title не длиннее 60 символов.
- [ ] Description не длиннее 160 символов.
- [ ] Ключевой запрос есть в title, description, H1, первом абзаце.
- [ ] Региональная привязка (Санкт-Петербург) присутствует.
- [ ] Текст без опечаток (визуальная проверка).
- [ ] Изображения имеют ALT.
- [ ] Изображения не битые.
- [ ] Внутренние ссылки работают.
- [ ] CTA в конце текста.
- [ ] FAQ-блок (если применимо) оформлен через Schema.org.
- [ ] Мобильная версия выглядит корректно.

## Техническая правка

- [ ] Backup создан перед правкой.
- [ ] Правка применена на сайте.
- [ ] Страница проверена с `?nocache=`.
- [ ] Нет ошибок в консоли браузера.
- [ ] Мобильная версия не сломана.
- [ ] Не затронуты соседние блоки.

## SEO

- [ ] URL понятный и короткий.
- [ ] Title уникальный для страницы.
- [ ] Description уникальный для страницы.
- [ ] Canonical указан (если нужно).
- [ ] Open Graph теги заполнены.
- [ ] Schema.org размечен корректно.

## Безопасность

- [ ] Не добавлены пароли/ключи в код.
- [ ] Правка не открывает уязвимость (XSS, SQL-инъекции).
- [ ] Служебные скрипты удалены после использования.

## Логирование

- [ ] Создана запись в `Logs/` или issue GitHub.
- [ ] Указано: что изменено, почему, когда, как проверить.
