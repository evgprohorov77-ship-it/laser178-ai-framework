# Post-Launch Backlog

## Версия

1.0.0

## Назначение

Идеи и задачи, которые не влияют на запуск сайта и откладываются до после релиза.

## Правила

- Здесь не ведётся активная работа до запуска `laser178.ru`.
- Каждая идея должна быть оценена по пользе для бизнеса.
- Перед переносом в активную работу нужен RFC.

## Запрещено до запуска

| ID | Идея | Почему отложено | Ожидаемая польза |
|----|------|-----------------|------------------|
| PL-1 | WordPress Adapter | Новый Adapter. Не критично для запуска, контент можно публиковать вручную. | Автоматизация публикации из Content/Drafts. |
| PL-2 | Telegram Adapter | Новый Adapter. Не влияет на сайт. | Интеграция уведомлений. |
| PL-3 | Cloudflare Adapter | Новый Adapter. Не критично. | CDN, защита. |
| PL-4 | Cron / автоаудит | Новый компонент. Можно делать вручную после запуска. | Регулярные проверки сайта. |
| PL-5 | LAOS Engine v2 | Новый Engine. Запрещено по feature freeze. | Не определена. |
| PL-6 | Memory / персистентность | Новый компонент. Не критично. | Долгосрочная память AI. |
| PL-7 | CLI для LAOS | Новый интерфейс. Не критично. | Удобство управления. |

## Как перенести в активную работу

1. Создать RFC в `RFC/`.
2. Обосновать пользу для бизнеса (KPI).
3. Получить approve от Owner.
4. Только после запуска сайта.
