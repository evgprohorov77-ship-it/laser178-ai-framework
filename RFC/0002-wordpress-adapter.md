# RFC-0002: WordPress Adapter

## Статус

Draft

## Автор

Hermes LAOS

## Дата

2026-07-13

## Краткое описание

Создать WordPress Adapter — единую точку взаимодействия LAOS с WordPress без использования REST API.

## Мотивация

Когда Policy Layer будет завершён, LAOS потребуется способ читать и изменять контент WordPress. REST API запрещён по соображениям безопасности. Нужен адаптер, который работает через прямой доступ к файлам/БД или через безопасный механизм, одобренный владельцем.

## Предлагаемое решение

1. Создать `AI/Adapters/WordPress/`.
2. Адаптер должен поддерживать:
   - чтение поста/страницы;
   - обновление meta (title, description, H1);
   - создание черновика;
   - публикацию по owner approval.
3. Адаптер не использует `/wp-json/`.
4. Все действия через адаптер требуют approval в production.

## Риски

- Прямой доступ к БД может быть опасен.
- Ошибки адаптера могут повредить контент.

## Митигация

- Backup Engine перед каждым действием.
- Verification Engine после каждого действия.
- Capability wordpress в staging перед production.

## Статус реализации

Not implemented. Ожидает завершения Security Layer.
