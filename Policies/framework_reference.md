# Framework Reference

## Формат

Каждый `Finding` обязан содержать поле `framework_reference` в формате:

```
{PREFIX}-{NNN} [§{SECTION}]
```

Примеры:

- `SEO-001 §2.1`
- `SEC-004 §1.2`
- `OPS-002 §3.1`

## Префиксы

| Префикс | Область | Пример |
|---------|---------|--------|
| FW | Общие правила Framework | FW-001 |
| SEO | Поисковая оптимизация | SEO-003 |
| OPS | Операции, backup, мониторинг | OPS-001 |
| SEC | Безопасность | SEC-002 |
| PERF | Производительность | PERF-004 |
| UX | Пользовательский опыт | UX-001 |
| STRUCT | Структура сайта, ссылки | STRUCT-001 |
| IMG | Изображения | IMG-001 |
| WP | WordPress-специфика | WP-001 |

## Отсутствие ссылки

Если `framework_reference` отсутствует или неизвестен:

1. Decision Engine возвращает `action: framework_improvement`.
2. Создаётся GitHub Issue «Framework Improvement: missing rule for {rule_id}».
3. Autofix запрещён.
4. Finding отправляется на ручную обработку.

## Расширение списка

При добавлении нового аудитора регистрируется новое правило в `Registry/rules.json` и соответствующий документ в `Framework/`.
