# Verification Policy

## Версия

1.0.0

## Цель

Убедиться, что изменение действительно устранило проблему и не создало новых.

## Когда проводится verification

После каждого автоматического или одобренного действия.

## Что проверяется

1. **HTTP 200.** Страница открывается.
2. **Отсутствие ошибок PHP.** Нет fatal error, warning, notice.
3. **Базовые SEO-метрики.**
   - Title не длиннее 60 символов.
   - Description не длиннее 160 символов.
   - H1 ровно один.
   - Canonical присутствует (если ожидается).
4. **Не появились новые проблемы.**
   - Нет новых битых ссылок.
   - Нет новых H1.
   - Нет новых изображений без alt.
5. **Исходная проблема устранена.**
   - Например, title стал короче 60 символов.
6. **Critical functions.**
   - wp-admin доступен.
   - Формы не сломаны.
   - Мобильная версия не разъезжается.

## Verification Engine

- Принимает на вход `finding`, `action_result`, `before_state`, `after_state`.
- Сравнивает before и after.
- Возвращает `status: success | failed | partial`.
- При `failed` — требует rollback.

## Rollback trigger

Verification проваливается, если:

- HTTP ≠ 200;
- появился новый H1 или H1 стал 0;
- title/description не изменились в нужную сторону;
- появились ошибки в HTML/JS;
- wp-admin недоступен;
- owner вручную отклонил результат.

## Логирование

Каждый результат verification записывается в `Logs/verification/`.
